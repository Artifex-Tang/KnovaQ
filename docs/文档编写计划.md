## 二、项目定位与系统概况（所有文档共用）

### 2.1 研究背景

**研究内容四：DARPA智能问答服务工具开发**

本部分研究内容联合军事科学院军事科学信息研究中心共同开展。

围绕构建面向DARPA智能问答服务工具，突破多源异构数据整合瓶颈，通过融合结构化知识管理与检索增强生成技术，打造具备高精度领域适应能力的**离线智能问答系统**。

### 2.2 系统标识

| 项目 | 值 |
|-----|---|
| 名称 | DARPA智能问答服务工具 |
| 标识 | DARPA-IQAS |
| 简称 | DARPA问答工具 |
| 版本号 | V1.0 |
| 研究内容 | 研究内容四 |
| 合作单位 | 军事科学院军事科学信息研究中心 |

### 2.3 三级架构设计

系统采用**"外挂知识库—RAG检索增强—交互式提示"三级架构**设计：

```
┌─────────────────────────────────────────────────────────────┐
│                    第三级：交互式提示词工程                    │
│   动态模板引擎 · 结构化约束机制 · 用户意图对齐 · 行为习惯适配  │
│         实现：用户意图/提问方式/行为习惯 ←→ DARPA文档知识      │
└────────────────────────────┬────────────────────────────────┘
                             │ 精准提示 + 检索结果
┌────────────────────────────▼────────────────────────────────┐
│                    第二级：RAG文档检索增强                     │
│     基于成熟框架领域适配 · 多文本特征融合 · 混合检索体系       │
│         向量检索 + 关键词检索 + 语义重排序 + 知识图谱          │
└────────────────────────────┬────────────────────────────────┘
                             │ 结构化知识片段
┌────────────────────────────▼────────────────────────────────┐
│                    第一级：外挂知识库                          │
│     非结构化军事文档 → 深度加工 → 语义化重构                   │
│     文档解析 · 智能分块 · 元数据标注 · 知识图谱构建            │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 三大核心模块

| 模块编号 | 模块名称 | 核心能力 | 功能描述 |
|---------|---------|---------|---------|
| M1 | 外挂知识库模块 | 深度加工与语义化重构 | 对非结构化军事文档进行解析、分块、元数据标注；支持PDF/Word/Excel/TXT/图片等多格式文档；实现知识库CRUD管理、文档上传与解析、智能分块管理、元数据过滤 |
| M2 | RAG文档检索增强模块 | 多文本特征融合混合检索 | 基于成熟开源框架(ragflow v0.18.0)进行领域适配；构建向量搜索、混合检索(向量+关键词)、相似度阈值调节、重排序、知识图谱、跨语言检索等多维度检索能力 |
| M3 | 交互式提示词工程模块 | 用户意图与知识精准对齐 | 通过动态模板引擎与结构化约束机制，实现聊天助手创建/管理、系统提示词配置、多轮上下文对话、流式响应、引用溯源、提示模板引擎等能力，将用户意图、提问方式、行为习惯与DARPA文档知识精准对齐 |

## 四、文档依赖关系与并行策略

```
                软件需求规格说明书 (Doc1)
                ┌─────────┬──────────┐
                │         │          │
                ▼         ▼          ▼
      软件设计说明书  系统测试大纲  软件用户手册
          (Doc2)     (Doc4)       (Doc3)
                │
                ▼
          系统测试报告 (Doc5)
```

**依赖说明：**
- Doc1（需求规格）是所有文档基础 → 最先写
- Doc2（设计说明）依赖 Doc1 需求定义
- Doc3（用户手册）依赖 Doc1 功能定义，与Doc2可并行
- Doc4（测试大纲）依赖 Doc1 需求，追踪每条需求到测试用例
- Doc5（测试报告）依赖 Doc4 测试定义 + 实际执行结果

**并行窗口分配：**

| 窗口 | 文档 | 前置条件 | 预估工作量 |
|-----|------|---------|----------|
| 窗口1 | Doc1 软件需求规格说明书 | 无，最先开始 | 最大（需求定义是基石） |
| 窗口2 | Doc2 软件设计说明书 | Doc1第3章完成 | 大（需要画图） |
| 窗口3 | Doc3 软件用户手册 | Doc1第3.2节完成 | 中（需要截图） |
| 窗口4 | Doc4 系统测试大纲 | Doc1第3章完成 | 中（可复用pytest代码） |
| 窗口5 | Doc5 系统测试报告 | Doc4完成 + 测试执行 | 小（填结果） |

**实际并行策略：**
- **阶段1**：窗口1写Doc1需求规格，其余窗口先读模板、准备框架
- **阶段2**：Doc1第3章(需求)写完 → 通知窗口2/3/4并行启动
- **阶段3**：Doc4测试大纲写完 → 窗口5开始Doc5测试报告
