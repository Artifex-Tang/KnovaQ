# KnovaQ Docker 部署工程 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the KnovaQ `docker/` directory — a deployable, multi-customer Docker environment combining ragflow 0.18.0 full with gaisoft-mes (Spring Boot) and gaisoft-ui (Vue 3/nginx).

**Architecture:** Single `docker-compose.yml` includes all ragflow 0.18.0 full services plus `gaisoft-server` and `gaisoft-frontend`. Per-customer config lives in `projects/<name>/`. Shell script `start.sh <project>` copies the customer's nginx config, then launches compose with the customer's `.env`. Build scripts copy external artifacts (`gaisoftmes.jar`, `html/`) into the `gaisoft/` runtime directory. Offline deployment uses `docker save`/`docker load` tar bundles.

**Tech Stack:** Docker Compose v2, ragflow 0.18.0 full images, `gaisoftmes` custom image (pre-existing), `nginx` image, bash scripts.

```yaml
services:

  # ── ragflow core ─────────────────────────────────────────────────────────
  ragflow:
    image: infiniflow/ragflow:${RAGFLOW_VERSION:-v0.18.0}
    container_name: ragflow-server
    depends_on:
      mysql:
        condition: service_healthy
      es01:
        condition: service_healthy
      minio:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      - TZ=${TIMEZONE:-Asia/Shanghai}
      - HF_ENDPOINT=${HF_ENDPOINT:-}
      - MACOS=${MACOS:-0}
    volumes:
      - ragflow_logs:/ragflow/logs
      - ragflow_upload:/ragflow/public/upload
    networks:
      - ragflow
    restart: always

  # ── Elasticsearch ────────────────────────────────────────────────────────
  es01:
    image: elasticsearch:${ELASTIC_VERSION:-8.11.3}
    container_name: ragflow-es-01
    environment:
      - node.name=ragflow-es-01
      - ELASTIC_PASSWORD=${ELASTIC_PASSWORD:-infini_rag_flow}
      - bootstrap.memory_lock=false
      - discovery.type=single-node
      - xpack.security.enabled=true
      - xpack.security.enrollment.enabled=false
      - xpack.security.http.ssl.enabled=false
      - cluster.routing.allocation.disk.watermark.low=85%
      - cluster.routing.allocation.disk.watermark.high=90%
      - cluster.routing.allocation.disk.watermark.flood_stage=95%
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - esdata01:/usr/share/elasticsearch/data
    networks:
      - ragflow
    healthcheck:
      test: ["CMD-SHELL", "curl -s http://localhost:9200/_cluster/health | grep -E '\"status\":\"(green|yellow)\"'"]
      interval: 10s
      timeout: 10s
      retries: 120
    restart: always

  # ── MySQL ────────────────────────────────────────────────────────────────
  mysql:
    image: mysql:${MYSQL_VERSION:-8.0.39}
    container_name: ragflow-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_PASSWORD:-infini_rag_flow}
      MYSQL_DATABASE: rag_flow
      MYSQL_CHARACTER_SET_SERVER: utf8mb4
      MYSQL_COLLATION_SERVER: utf8mb4_unicode_ci
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init:/docker-entrypoint-initdb.d   # equipment_iqas.sql runs on first boot only
    networks:
      - ragflow
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-uroot", "-p${MYSQL_PASSWORD:-infini_rag_flow}"]
      interval: 10s
      timeout: 10s
      retries: 3
    restart: always

  # ── MinIO ────────────────────────────────────────────────────────────────
  minio:
    image: quay.io/minio/minio:${MINIO_VERSION:-RELEASE.2023-12-20T01-00-02Z}
    container_name: ragflow-minio
    environment:
      MINIO_ROOT_USER: ${MINIO_USER:-rag_flow}
      MINIO_ROOT_PASSWORD: ${MINIO_PASSWORD:-infini_rag_flow}
    command: server --console-address ":9001" /data
    volumes:
      - minio_data:/data
    networks:
      - ragflow
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:9000/minio/health/live"]
      interval: 10s
      timeout: 10s
      retries: 3
    restart: always

  # ── Redis (Valkey) ───────────────────────────────────────────────────────
  redis:
    image: valkey/valkey:${VALKEY_VERSION:-8}
    container_name: ragflow-redis
    command: valkey-server --requirepass ${REDIS_PASSWORD:-infini_rag_flow}
    volumes:
      - redis_data:/data
    networks:
      - ragflow
    healthcheck:
      test: ["CMD", "valkey-cli", "-a", "${REDIS_PASSWORD:-infini_rag_flow}", "ping"]
      interval: 10s
      timeout: 10s
      retries: 3
    restart: always

  # ── gaisoft-server (Spring Boot) ─────────────────────────────────────────

    image: gaisoftmes
    container_name: equipment-server
    depends_on:

        condition: service_healthy
      redis:
        condition: service_healthy
    volumes:
      - ./gaisoft/jar:/usr/soft
      - ./gaisoft/uploadfile:/ragflow/uploadPath
    networks:
      - ragflow
    ports:
      - "${GAISOFT_SERVER_PORT:-8080}:8080"

    image: nginx

      - gaisoft-server
    volumes:
      - ./gaisoft/nginx/html:/usr/share/nginx/html
      - ./gaisoft/nginx/logs:/var/log/nginx
      - ./gaisoft/nginx/conf.d:/etc/nginx/conf.d
    networks:
      - ragflow
    ports:
      - "${GAISOFT_FRONTEND_PORT:-8899}:80"
    restart: always

networks:
  ragflow:
    driver: bridge

  esdata01:
  mysql_data:
  minio_data:

  ragflow_upload:

```bash
#!/usr/bin/env bash
set -euo pipefail

PROJECT="${1:-}"
if [ -z "$PROJECT" ]; then
    echo "Usage: $0 <project>"
    echo "Available projects:"
    ls "$(dirname "$0")/../projects/" | grep -v '^_'
    exit 1
fi

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DOCKER_DIR="$(dirname "$SCRIPT_DIR")"
PROJECT_DIR="$DOCKER_DIR/projects/$PROJECT"

if [ ! -d "$PROJECT_DIR" ]; then
    echo "Error: project '$PROJECT' not found at $PROJECT_DIR"
    exit 1
fi

# Copy this customer's nginx config into the runtime directory
cp "$PROJECT_DIR/nginx/default.conf" \
   "$DOCKER_DIR/gaisoft/nginx/conf.d/default.conf"
echo "✓ nginx config copied from projects/$PROJECT/"

# Launch compose with this customer's env overrides
cd "$DOCKER_DIR"
docker compose --env-file "$PROJECT_DIR/.env" up -d

echo "✓ KnovaQ started for project: $PROJECT"
```

### Task 9: offline-save.sh + offline-load.sh

**Files:**

- Create: `docker/scripts/offline-load.sh`

- [ ] **Step 1: Create docker/scripts/offline-save.sh**

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DOCKER_DIR="$(dirname "$SCRIPT_DIR")"
IMAGES_DIR="$DOCKER_DIR/images"

mkdir -p "$IMAGES_DIR"

cd "$DOCKER_DIR"

# Get full list of images from compose (includes all service images)
IMAGES=$(docker compose config --images 2>/dev/null)

if [ -z "$IMAGES" ]; then
    echo "Error: could not read images from docker-compose.yml"
    exit 1
fi

for IMAGE in $IMAGES; do
    # Convert image name to safe filename: replace / and : with _
    FILENAME="${IMAGE//\//_}"
    FILENAME="${FILENAME//:/_}.tar"
    echo "Saving $IMAGE → images/$FILENAME"
    docker save "$IMAGE" -o "$IMAGES_DIR/$FILENAME"
done

echo ""
echo "✓ All images saved to docker/images/"
echo "  Total size: $(du -sh "$IMAGES_DIR" | cut -f1)"
echo ""
echo "Next: tar -czf knovaq-offline.tar.gz docker/ (then copy to customer machine)"
```

- [ ] **Step 2: Create docker/scripts/offline-load.sh**

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DOCKER_DIR="$(dirname "$SCRIPT_DIR")"
IMAGES_DIR="$DOCKER_DIR/images"

if [ ! -d "$IMAGES_DIR" ] || [ -z "$(ls "$IMAGES_DIR"/*.tar 2>/dev/null)" ]; then
    echo "Error: no .tar files found in $IMAGES_DIR"
    echo "Copy offline image tarballs to docker/images/ first"
    exit 1
fi

for TAR in "$IMAGES_DIR"/*.tar; do
    echo "Loading $(basename "$TAR") ..."
    docker load -i "$TAR"
done

echo ""
echo "✓ All images loaded"
echo "Now run: ./scripts/start.sh <project>"
```

## Architecture

```
docker/
├── docker-compose.yml        ragflow 0.18.0 full + gaisoft-server + gaisoft-frontend
├── .env                      global defaults (image versions, default passwords/ports)
├── projects/<customer>/      per-customer .env overrides + nginx/default.conf
├── gaisoft/
│   ├── jar/gaisoftmes.jar    Spring Boot jar (updated by build-mes.sh)
│   ├── uploadfile/           persistent upload storage
│   └── nginx/                runtime nginx config, html, logs
├── init/
│   └── equipment_iqas.sql    gaisoft-mes DB init (runs once on first MySQL boot)
└── scripts/                  start, stop, build, offline scripts
```

## Network

All services share the `ragflow` bridge network (Docker names it `docker_ragflow` because the compose lives in the `docker/` directory). gaisoft-server connects to ragflow-mysql:3306 and ragflow-redis:6379 using credentials from `.env`.

## DB Init Behavior

`docker/init/equipment_iqas.sql` is mounted to MySQL's `/docker-entrypoint-initdb.d/`. MySQL runs it exactly once — when the data volume is first created. Re-running init requires deleting the `mysql_data` Docker volume (`docker volume rm docker_mysql_data`), which destroys all data.

## gaisoft-server Environment (from gaisoft-mes config)

| Setting | Value |
|---------|-------|
| MySQL host | `ragflow-mysql:3306` |
| MySQL database | `equipment_iqas` |
| Redis host | `ragflow-redis:6379` |
| Redis database | `8` |
| Upload path | `/ragflow/uploadPath` (mounted from `gaisoft/uploadfile/`) |
```
