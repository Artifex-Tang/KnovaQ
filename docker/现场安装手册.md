# 知枢 · KnovaQ 现场安装手册

> 照着一步步复制命令执行即可。遇到报错，看最后【第 9 节 常见问题】。

---

## 第 0 步：先准备好这些

**一台 Linux 服务器**（推荐 Ubuntu 22.04 / 24.04，64 位）：
- 内存 **≥ 16 GB**（实测 30 GB 最稳，低于 8 GB 会崩）
- 硬盘空闲 **≥ 30 GB**
- 已装好 Docker（命令 `docker compose version` 能正常输出版本）

**你手上应有两个压缩包**：
1. `KnovaQ_镜像包.tar.gz`（很大，约 9 GB，里面是 7 个 Docker 镜像）
2. `KnovaQ_现场部署包.zip`（小，约 300 MB，配置+脚本+本手册+测试报告）

> 如果服务器**以前装过、镜像还在**，可跳过镜像包，只用部署包（见第 4 步说明）。

---

## 第 1 步：把两个包上传到服务器

在**你自己的电脑**上执行（把 `服务器IP` 换成实际地址）：

```bash
scp KnovaQ_镜像包.tar.gz KnovaQ_现场部署包.zip root@服务器IP:/app/
```

> 没有 scp 就用 WinSCP、Xftp 等工具，传到服务器 `/app/` 目录。

---

## 第 2 步：登录服务器并解压

```bash
ssh root@服务器IP

cd /app
unzip KnovaQ_现场部署包.zip -d KnovaQ        # 解压出 KnovaQ/docker/...
mkdir -p KnovaQ/docker/images
tar -xzf KnovaQ_镜像包.tar.gz -C KnovaQ/docker/images/   # 镜像 tar 解到 images/
cd KnovaQ/docker
chmod +x scripts/*.sh
```

完成后 `ls scripts/` 应能看到 `start.sh`、`stop.sh`、`offline-load.sh`、`verify.sh` 等。

---

## 第 3 步：设置一个内核参数（必做，一次即可）

Elasticsearch 必需，不设会起不来：

```bash
sudo sysctl -w vm.max_map_count=262144
grep -q vm.max_map_count /etc/sysctl.conf || echo 'vm.max_map_count=262144' | sudo tee -a /etc/sysctl.conf
```

---

## 第 4 步：加载镜像

```bash
cd /app/KnovaQ/docker
bash scripts/offline-load.sh
```

看到 **7 个镜像全部 Loaded / 校验通过** 即可。

> **服务器以前装过、镜像没动**：跳过本步（镜像已在 Docker 里）。

---

## 第 5 步：一键启动（自动建库 + 自检）

```bash
bash scripts/start.sh
```

脚本会自动：起全栈 → 等 ragflow 就绪 → 建库/种子数据 → **自动跑自检 verify**。
全程几分钟，日志实时打印，也存到 `logs/deploy-*.log`。

---

## 第 6 步：确认装好了

启动末尾会自动输出自检结果，看到类似：

```
verify: 16/16 PASS
```

就说明 **装成功了**。若有 FAIL，看 `logs/verify-*.log` 和第 9 节。

也可随时手动复检：
```bash
bash scripts/verify.sh
```

---

## 第 7 步：配置大模型 Key（客户自己的 Key）

系统自带**本地离线嵌入模型**（免 Key），但**对话**需要客户自己的大模型 Key。

1. 浏览器打开知识库后台：`http://服务器IP:8070`
2. 登录（首次按页面提示）
3. 右上角头像 →【模型提供商 / Model providers】
4. 找到要用的厂商（如 **ZHIPU-AI 智谱**、**DeepSeek**、**SiliconFlow 硅基流动**），填入 **API Key**，保存
5. 回到【系统模型设置】，把 **聊天模型 Chat** 选成刚配好的模型（如 `glm-4-flash` / `deepseek-chat`），保存

> 嵌入模型默认已是本地 `bge`，无需改。只配对话 Key 即可。

---

## 第 8 步：访问系统

| 用途 | 地址 | 说明 |
|---|---|---|
| **主系统**（DARPA 智能问答工具） | `http://服务器IP:8899` | 业务前端 |
| 知识库后台（ragflow） | `http://服务器IP:8070` | 配模型/管知识库 |
| 数据库（如需用 Workbench 连） | `服务器IP:5455` | 账号 `root` 密码见 `.env` |

> 云服务器记得在**安全组/防火墙**放行 `8899`、`8070`（和需要的 `5455`）的入方向 TCP。

---

## 第 9 步：常见问题排查

| 现象 | 原因 | 解决 |
|---|---|---|
| ES 容器反复重启 | 没设内核参数 | 重做第 3 步，再 `bash scripts/stop.sh && bash scripts/start.sh` |
| 起栈很慢/卡 | 内存不足 | 确认 ≥16 GB；`free -h` 查看 |
| 页面打不开 | 防火墙/安全组没放行 | 放行 8899/8070 入方向；本机带代理时换无代理机器测 |
| 登录提示「验证码已失效」 | 验证码服务异常 | 重启后端：`docker restart equipment-server` |
| 问答提示「登录状态已过期/未知错误」 | 模型 Key 没配 | 做第 7 步，配对话模型 Key |
| 上传文档一直「解析中」不出结果 | 对话/嵌入异常或一次传太多 | 单次别批量传几百个；确认嵌入模型正常；`bash scripts/verify.sh` |
| 想重置重装 | — | `docker compose down -v`（**会清空所有数据**）后重做第 5 步 |

---

## 第 10 步：安全加固（上线前务必做）

1. **改默认密码**：编辑 `docker/.env`，把 `infini_rag_flow` 等默认口令改掉，再 `down -v` 重启。
2. **收窄访问来源**：安全组把 `5455/8899/8070` 来源限制到办公 IP，**不要留 `0.0.0.0/0`**。
3. **生产开验证码**：如需登录验证码，确认 `sys.account.captchaEnabled=true`。
